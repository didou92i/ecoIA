# Politique de sécurité

## Versions prises en charge

Tant que le projet est en série `0.x`, seule la dernière version publiée reçoit des correctifs de
sécurité.

## Signaler une vulnérabilité

Sur le [dépôt GitHub public](https://github.com/didou92i/ecoIA), utilisez l’onglet **Security**, puis
**Report a vulnerability**,
afin d’ouvrir un signalement privé. N’incluez pas de secret, de donnée personnelle ou de preuve
offensive dans une issue publique. Si le signalement privé n’est pas encore activé, ouvrez une issue
publique minimale demandant un canal privé, sans divulguer les détails techniques.

Indiquez la version, le navigateur, le composant affecté, l’impact supposé et une reproduction
non destructive. Le mainteneur accusera réception via le canal GitHub, qualifiera le problème et
coordonnera la publication d’un correctif. Aucun délai fixe n’est promis avant que le projet ne
dispose d’une équipe de maintenance formelle.

## Périmètre

Les tests doivent rester défensifs, locaux et non destructifs. Ne testez pas les plateformes IA ou
les comptes d’autres personnes sans autorisation explicite. Ne joignez jamais un prompt réel ou une
session de navigateur à un rapport.
